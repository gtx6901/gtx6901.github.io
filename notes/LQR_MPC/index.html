# 从 LQR 到 MPC，再到 NMPC

# —— 从最优控制到现代机器人控制方法

# 一、引言

对于现代的很多控制系统，我们已经不再满足于系统能否安全正常运行，而更关心它们的运行是否高效，以及是否达到了 效率和能耗的最优。由此，我们引入了最优控制理论。最优控制不仅关心如何使系统保持稳定，更关心如何在保持稳定的同时，使得系统的某些性能指标达到最优。

在最优控制理论的发展过程中，针对线性系统与二次型性能指标的优化问题，诞生了一种最具代表性且具有解析解的经典方法 —— 线性二次型调节器（Linear Quadratic Regulator, LQR）。LQR 不仅为最优控制提供了严谨的数学基础，也为后续模型预测控制 (MPC) 的发展奠定了理论基石。

事实上，LQR 与 MPC 之间存在着深刻的理论联系。LQR 为线性系统提供了具有解析解的无限时域最优控制律，而 MPC 则通过滚动时域优化，将最优控制思想扩展至有限时域，并能够显式处理系统约束。

在 无约束条件下，当预测时域趋于无穷，且终端权重取为黎卡提方程的解时，MPC 将退化为 LQR。因此，可以认为 LQR 是 MPC 在特定条件下的特例，而 MPC 则是 LQR 在约束条件和在线优化框架下的推广。这种关系构成了现代最优控制理论的重要桥梁，并为机器人与自动驾驶系统中的控制算法设计提供了统一的理论基础。

# 二、离散型线性二次系统（离散型 LQR）

首先，让我们从最简单的例子出发，考虑一个离散型线性系统：

J=∑k=0∞(xkTQxk+ukTRuk)J = \sum_{k=0}^{\infty} \left( x_k^T Q x_k + u_k^T R u_k \right) J=k=0∑∞​(xkT​Qxk​+ukT​Ruk​)

  • 系统是线性的
  • 代价是二次型

这就是经典的 LQR 反馈形式。

# 2、离散代数 Riccati 方程（DARE）

把最优控制律再代回 Bellman 方程，可以得到 PPP 需要满足：

P=Q+ATPA−ATPB(R+BTPB)−1BTPAP = Q + A^TPA - A^TPB\left(R+B^TPB\right)^{-1}B^TPA P=Q+ATPA−ATPB(R+BTPB)−1BTPA

这就是离散代数黎卡提方程（DARE）。

求解流程通常是：

  1. 给定系统矩阵 A,BA,BA,B 和权重 Q,RQ,RQ,R
  2. 数值求解 DARE 得到 PPP
  3. 计算反馈增益 KKK
  4. 在线控制使用 uk=−Kxku_k=-Kx_kuk​=−Kxk​

# 三、连续型线性二次系统（连续型 LQR）

离散型 LQR 常用于数字控制器实现，而连续型 LQR 更像理论 “母版”。

# 1、系统模型与目标函数

考虑连续时间线性系统：

x˙(t)=Ax(t)+Bu(t)\dot{x}(t) = A x(t) + B u(t) x˙(t)=Ax(t)+Bu(t)

目标函数写为：

J=∫0∞(xTQx+uTRu)dtJ = \int_0^{\infty} \left(x^TQx + u^TRu\right)dt J=∫0∞​(xTQx+uTRu)dt

其中仍有 Q⪰0,R≻0Q\succeq0, R\succ0Q⪰0,R≻0。

# 2、从 HJB 到 CARE

设最优值函数：

V(x)=xTPxV(x)=x^TPx V(x)=xTPx

其中 P=PT⪰0P=P^T\succeq0P=PT⪰0。这个设定背后的理由与离散情形一致：在线性系统 + 二次代价结构下，值函数对 Bellman/HJB 算子是二次型闭包。

先给出 HJB（Hamilton-Jacobi-Bellman）条件。对无限时域连续系统

x˙=f(x,u),ℓ(x,u)=xTQx+uTRu\dot{x}=f(x,u),\qquad \ell(x,u)=x^TQx+u^TRu x˙=f(x,u),ℓ(x,u)=xTQx+uTRu

最优值函数满足

0=min⁡u(ℓ(x,u)+∇V(x)Tf(x,u))0 = \min_u \Big(\ell(x,u)+\nabla V(x)^T f(x,u)\Big) 0=umin​(ℓ(x,u)+∇V(x)Tf(x,u))

这条式子可以理解为：在最优策略下，“瞬时代价” + “未来最优代价变化率” 的和达到最小并等于 0。

把中括号内定义为哈密顿函数（Hamiltonian）：

H(x,u,∇V)=ℓ(x,u)+∇V(x)Tf(x,u)\mathcal{H}(x,u,\nabla V)=\ell(x,u)+\nabla V(x)^Tf(x,u) H(x,u,∇V)=ℓ(x,u)+∇V(x)Tf(x,u)

它的物理意义是 “当前这一瞬间的总成本密度”，包含两部分：

  • ℓ(x,u)\ell(x,u)ℓ(x,u)：眼前付出的控制与状态代价
  • ∇VTf\nabla V^Tf∇VTf：状态沿动力学演化时，剩余最优代价的变化率

对于 LQR，f(x,u)=Ax+Buf(x,u)=Ax+Buf(x,u)=Ax+Bu，且

∇V(x)=2Px\nabla V(x)=2Px ∇V(x)=2Px

所以

∇VTf=2xTP(Ax+Bu)\nabla V^Tf = 2x^TP(Ax+Bu) ∇VTf=2xTP(Ax+Bu)

等价写法是把这部分记成

V˙=xT(ATP+PA)x+2xTPBu\dot{V}=x^T(A^TP+PA)x+2x^TPBu V˙=xT(ATP+PA)x+2xTPBu

因此 HJB 可写为：

0=min⁡u(xTQx+uTRu+xT(ATP+PA)x+2xTPBu)0 = \min_u \left(x^TQx + u^TRu + x^T(A^TP+PA)x+2x^TPBu\right) 0=umin​(xTQx+uTRu+xT(ATP+PA)x+2xTPBu)

对 uuu 求偏导并令其为零：

∂H∂u=2Ru+2BTPx=0\frac{\partial \mathcal{H}}{\partial u}=2Ru+2B^TPx=0 ∂u∂H​=2Ru+2BTPx=0

得到最优控制律：

u∗=−R−1BTPx=−Kxu^* = -R^{-1}B^TPx = -Kx u∗=−R−1BTPx=−Kx

代回 HJB：

0=xT(Q+ATP+PA−PBR−1BTP)x0=x^T\Big(Q+A^TP+PA-PBR^{-1}B^TP\Big)x 0=xT(Q+ATP+PA−PBR−1BTP)x

由于该式对任意 xxx 都成立，括号内矩阵必须为零，于是得到连续代数黎卡提方程（CARE）：

ATP+PA−PBR−1BTP+Q=0A^TP + PA - PBR^{-1}B^TP + Q = 0 ATP+PA−PBR−1BTP+Q=0

于是连续 LQR 增益为：

K=R−1BTPK = R^{-1}B^TP K=R−1BTP

CARE 的作用可以理解为：它是 “把最优控制问题变成矩阵方程” 的桥梁。只要解出稳定化解 PPP，就能立刻得到最优反馈增益 KKK。因此连续 LQR 的工程流程通常是：先解 CARE，再上线性状态反馈 u=−Kxu=-Kxu=−Kx。

# 3、离散与连续的关系

实际控制器常以采样周期 Δt\Delta tΔt 运行。将连续系统离散化可得：

Ad=eAΔt,Bd=∫0ΔteAτB dτA_d=e^{A\Delta t}, \quad B_d=\int_0^{\Delta t}e^{A\tau}B\,d\tau Ad​=eAΔt,Bd​=∫0Δt​eAτBdτ

然后直接在离散域做 LQR 设计，这也是机器人控制里最常见的流程。

# 四、模型预测控制（MPC）

# 0、先说清楚：MPC 到底在做什么

MPC（Model Predictive Control）不是一条固定控制公式，而是一种 “在线滚动优化” 方法：

  1. 用模型预测未来 NNN 步状态
  2. 在所有可行控制序列中找最优
  3. 只执行第一步控制
  4. 下一采样时刻重新优化

这 4 步不断循环，就是 MPC 的核心。

# 1、从线性离散系统开始

系统模型：

xk+1=Axk+Bukx_{k+1}=Ax_k+Bu_k xk+1​=Axk​+Buk​

常用的跟踪型代价函数：

J=∑i=0N−1[(xi−xref)TQ(xi−xref)+uiTRui]+(xN−xref)TP(xN−xref)J = \sum_{i=0}^{N-1}\left[(x_i-x_{ref})^TQ(x_i-x_{ref}) + u_i^TRu_i\right] + (x_N-x_{ref})^TP(x_N-x_{ref}) J=i=0∑N−1​[(xi​−xref​)TQ(xi​−xref​)+uiT​Rui​]+(xN​−xref​)TP(xN​−xref​)

解释：

  • QQQ 大：更在乎跟踪误差
  • RRR 大：更在乎控制平滑
  • PPP：保证预测末端别 “散” 掉

# 2、把未来状态一步一步展开

从当前状态 x0x_0x0​ 出发：

x1=Ax0+Bu0x_1 = Ax_0 + Bu_0 x1​=Ax0​+Bu0​

x2=A2x0+ABu0+Bu1x_2 = A^2x_0 + ABu_0 + Bu_1 x2​=A2x0​+ABu0​+Bu1​

x3=A3x0+A2Bu0+ABu1+Bu2x_3 = A^3x_0 + A^2Bu_0 + ABu_1 + Bu_2 x3​=A3x0​+A2Bu0​+ABu1​+Bu2​

归纳可得：

xi=Aix0+∑j=0i−1Ai−1−jBujx_i = A^ix_0 + \sum_{j=0}^{i-1}A^{i-1-j}Bu_j xi​=Aix0​+j=0∑i−1​Ai−1−jBuj​

这一步是线性 MPC 推导的核心：未来状态被显式写成 “当前状态 + 控制序列” 的函数。

  1. 写出 x1,x2,x3x_1, x_2, x_3x1​,x2​,x3​；
  2. 构造预测模型 X=Ax0+BUX = \mathcal{A}x_0 + \mathcal{B}UX=Ax0​+BU
  3. 写出 MPC 的二次型优化目标函数。

# 五、非线性模型预测控制（NMPC）

# 0、为什么线性 MPC 会走向 NMPC

现在开始最关键的问题：为什么线性 MPC 会变成 NMPC？

答案很简单：因为真实系统往往不是线性的。

线性 MPC 建立在：

xk+1=Axk+Bukx_{k+1}=Ax_k+Bu_k xk+1​=Axk​+Buk​

但很多机器人不是这样。以差速底盘（独轮车模型）为例：

p˙x=vcos⁡θ,p˙y=vsin⁡θ,θ˙=ω\dot p_x=v\cos\theta,\quad \dot p_y=v\sin\theta,\quad \dot\theta=\omega p˙​x​=vcosθ,p˙​y​=vsinθ,θ˙=ω

离散化后：

px,k+1=px,k+Δt vkcos⁡θkp_{x,k+1}=p_{x,k}+\Delta t\,v_k\cos\theta_k px,k+1​=px,k​+Δtvk​cosθk​

py,k+1=py,k+Δt vksin⁡θkp_{y,k+1}=p_{y,k}+\Delta t\,v_k\sin\theta_k py,k+1​=py,k​+Δtvk​sinθk​

θk+1=θk+Δt ωk\theta_{k+1}=\theta_k+\Delta t\,\omega_k θk+1​=θk​+Δtωk​

显然它不是 Axk+BukAx_k+Bu_kAxk​+Buk​，因此更一般地写成：

xk+1=f(xk,uk)x_{k+1}=f(x_k,u_k) xk+1​=f(xk​,uk​)

这就是 NMPC 的起点。

# 1、NMPC 的标准优化问题

和线性 MPC 很像，只是模型变成非线性：

min⁡{xk,uk}J=∑k=0N−1[(xk−xkref)TQ(xk−xkref)+(uk−ukref)TR(uk−ukref)]+(xN−xNref)TP(xN−xNref)\min_{\{x_k,u_k\}}J= \sum_{k=0}^{N-1}\Big[(x_k-x_k^{ref})^TQ(x_k-x_k^{ref})+(u_k-u_k^{ref})^TR(u_k-u_k^{ref})\Big] +(x_N-x_N^{ref})^TP(x_N-x_N^{ref}) {xk​,uk​}min​J=k=0∑N−1​[(xk​−xkref​)TQ(xk​−xkref​)+(uk​−ukref​)TR(uk​−ukref​)]+(xN​−xNref​)TP(xN​−xNref​)

约束：

xk+1=f(xk,uk),k=0,…,N−1x_{k+1}=f(x_k,u_k),\quad k=0,\dots,N-1 xk+1​=f(xk​,uk​),k=0,…,N−1

umin≤uk≤umax,xmin≤xk≤xmaxu_{min}\le u_k\le u_{max},\qquad x_{min}\le x_k\le x_{max} umin​≤uk​≤umax​,xmin​≤xk​≤xmax​

h(xk,uk)≤0h(x_k,u_k)\le0 h(xk​,uk​)≤0

因此 NMPC 的标准形式是一个非线性规划（NLP）。

# 2、NMPC 和线性 MPC 最本质的区别

不是 “多了个 N”，也不是 “换了个求解器”，而是：

  • 在线性 MPC 里，未来状态可显式消元：X=Ax0+BUX=\mathcal{A}x_0+\mathcal{B}UX=Ax0​+BU
  • 在 NMPC 里，一般不能写成这种线性显式形式

所以线性 MPC 通常是 QP，NMPC 通常是 NLP。

# 3、先用 Single Shooting 理解 NMPC

Single Shooting 是最直观的入门方式：

  1. 把控制序列 U={u0,u1,…,uN−1}U=\{u_0,u_1,\dots,u_{N-1}\}U={u0​,u1​,…,uN−1​} 作为优化变量
  2. 给定当前状态 x0x_0x0​，按动力学递推：

x1=f(x0,u0),  x2=f(x1,u1),  …,  xN=f(xN−1,uN−1)x_1=f(x_0,u_0),\;x_2=f(x_1,u_1),\;\dots,\;x_N=f(x_{N-1},u_{N-1}) x1​=f(x0​,u0​),x2​=f(x1​,u1​),…,xN​=f(xN−1​,uN−1​)

  1. 于是状态成为控制序列的函数：

xk=ϕk(x0,u0,…,uk−1)x_k=\phi_k(x_0,u_0,\dots,u_{k-1}) xk​=ϕk​(x0​,u0​,…,uk−1​)

  1. 代价函数变成 J(U)J(U)J(U)，对 UUU 做数值优化

关键点在于：J(U)J(U)J(U) 一般不再是二次函数，而是非线性函数。

# 4、差速底盘的 NMPC 求解（运动学模型）

# 4.1 状态、控制与模型

状态：

xk=[px,kpy,kθk]x_k= \begin{bmatrix} p_{x,k}\\ p_{y,k}\\ \theta_k \end{bmatrix} xk​=⎣⎢⎡​px,k​py,k​θk​​⎦⎥⎤​

控制：

uk=[vkωk]u_k= \begin{bmatrix} v_k\\ \omega_k \end{bmatrix} uk​=[vk​ωk​​]

离散模型：

px,k+1=px,k+Δt vkcos⁡θkp_{x,k+1}=p_{x,k}+\Delta t\,v_k\cos\theta_k px,k+1​=px,k​+Δtvk​cosθk​

py,k+1=py,k+Δt vksin⁡θkp_{y,k+1}=p_{y,k}+\Delta t\,v_k\sin\theta_k py,k+1​=py,k​+Δtvk​sinθk​

θk+1=θk+Δt ωk\theta_{k+1}=\theta_k+\Delta t\,\omega_k θk+1​=θk​+Δtωk​

# 4.2 代价函数设计

J=∑k=0N−1[(xk−xkref)TQ(xk−xkref)+ukTRuk]+(xN−xNref)TP(xN−xNref)J= \sum_{k=0}^{N-1}\Big[(x_k-x_k^{ref})^TQ(x_k-x_k^{ref})+u_k^TRu_k\Big] +(x_N-x_N^{ref})^TP(x_N-x_N^{ref}) J=k=0∑N−1​[(xk​−xkref​)TQ(xk​−xkref​)+ukT​Ruk​]+(xN​−xNref​)TP(xN​−xNref​)

若需要更平滑控制，可加入：

∑k=0N−2(uk+1−uk)TS(uk+1−uk)\sum_{k=0}^{N-2}(u_{k+1}-u_k)^TS(u_{k+1}-u_k) k=0∑N−2​(uk+1​−uk​)TS(uk+1​−uk​)

# 4.3 约束设计

输入约束：

vmin≤vk≤vmax,ωmin≤ωk≤ωmaxv_{min}\le v_k\le v_{max},\qquad \omega_{min}\le\omega_k\le\omega_{max} vmin​≤vk​≤vmax​,ωmin​≤ωk​≤ωmax​

障碍约束（圆形障碍示例）：

hobs(xk)=(robs+rsafe)2−((px,k−pxobs)2+(py,k−pyobs)2)≤0h_{obs}(x_k) =(r_{obs}+r_{safe})^2 - ( (p_{x,k}-p_x^{obs})^2+(p_{y,k}-p_y^{obs})^2 ) \le 0 hobs​(xk​)=(robs​+rsafe​)2−((px,k​−pxobs​)2+(py,k​−pyobs​)2)≤0

# 4.4 NLP 转录（Single Shooting）

以控制序列 UUU 为决策变量：

min⁡UJ(U)\min_U J(U) Umin​J(U)

subject to

xk+1=f(xk,uk),x0=x^(t)x_{k+1}=f(x_k,u_k),\quad x_0=\hat{x}(t) xk+1​=f(xk​,uk​),x0​=x^(t)

以及输入 / 障碍等约束。

# 4.5 在线滚动求解流程

每个采样时刻执行：

  1. 读取当前状态估计 x^(t)\hat{x}(t)x^(t)
  2. 用上一时刻最优解平移作 warm-start
  3. 用系统模型滚动预测状态轨迹
  4. 计算总代价与约束
  5. 用 SQP / IPOPT / ACADOS 迭代求解 NLP
  6. 取第一步控制 u0∗u_0^*u0∗​ 下发给底盘
  7. 时域前移，进入下一轮优化

这就是 NMPC 的 receding horizon 闭环。

# 5、为什么 NMPC 通常没有解析解

NMPC 最终面对的是

min⁡UJ(U)\min_U J(U) Umin​J(U)

但 J(U)J(U)J(U) 由非线性递推 xk+1=f(xk,uk)x_{k+1}=f(x_k,u_k)xk+1​=f(xk​,uk​) 隐式定义，还叠加了非线性约束。通常没有闭式解，只能靠数值优化迭代求近似最优解。

# 6、线性化是连接线性 MPC 与 NMPC 的桥梁

对非线性系统在工作点 (x∗,u∗)(x^*,u^*)(x∗,u∗) 附近做一阶泰勒展开：

f(x,u)≈f(x∗,u∗)+∂f∂x∣x∗,u∗(x−x∗)+∂f∂u∣x∗,u∗(u−u∗)f(x,u)\approx f(x^*,u^*)+ \left.\frac{\partial f}{\partial x}\right|_{x^*,u^*}(x-x^*)+ \left.\frac{\partial f}{\partial u}\right|_{x^*,u^*}(u-u^*) f(x,u)≈f(x∗,u∗)+∂x∂f​∣∣∣∣∣​x∗,u∗​(x−x∗)+∂u∂f​∣∣∣∣∣​x∗,u∗​(u−u∗)

设 δx=x−x∗,δu=u−u∗\delta x=x-x^*,\delta u=u-u^*δx=x−x∗,δu=u−u∗，得到局部线性时变模型：

δxk+1≈Akδxk+Bkδuk\delta x_{k+1}\approx A_k\delta x_k+B_k\delta u_k δxk+1​≈Ak​δxk​+Bk​δuk​

其中

Ak=∂f∂x∣x∗,u∗,Bk=∂f∂u∣x∗,u∗A_k=\left.\frac{\partial f}{\partial x}\right|_{x^*,u^*},\qquad B_k=\left.\frac{\partial f}{\partial u}\right|_{x^*,u^*} Ak​=∂x∂f​∣∣∣∣∣​x∗,u∗​,Bk​=∂u∂f​∣∣∣∣∣​x∗,u∗​

这也是很多快 NMPC 方法（Sequential QP / RTI）的核心思想：每个采样时刻在线性化，再解局部 QP。

# 六、LQR 与 MPC 的联系

现在，在完成对 LQR 和 MPC 的全面了解后，让我们回过头去观察两者的异同。

# 1. 统一的最优控制框架

LQR 与 MPC 均源于线性系统的二次型性能指标优化问题。对于离散时间系统：

xk+1=Axk+Bukx_{k+1} = A x_k + B u_k xk+1​=Axk​+Buk​

两者的目标函数分别为：

LQR（无限时域）：

J=∑k=0∞(xkTQxk+ukTRuk)J = \sum_{k=0}^{\infty} \left( x_k^T Q x_k + u_k^T R u_k \right) J=k=0∑∞​(xkT​Qxk​+ukT​Ruk​)

MPC（有限时域）：

J=∑k=0N−1(xkTQxk+ukTRuk)+xNTPxNJ = \sum_{k=0}^{N-1} \left( x_k^T Q x_k + u_k^T R u_k \right) + x_N^T P x_N J=k=0∑N−1​(xkT​Qxk​+ukT​Ruk​)+xNT​PxN​

可以看出，MPC 是对 LQR 无限时域问题的截断与近似。

# 2. 终端权重与黎卡提解

当终端权重矩阵取为离散代数黎卡提方程（DARE）的解时：

P=PLQRP = P_{LQR} P=PLQR​

终端代价满足：

xNTPxNx_N^T P x_N xNT​PxN​

该项等价于系统从预测末端开始，在无限时域内由最优 LQR 控制器继续控制所产生的最小剩余代价（Cost-to-Go）。因此，终端项实际上是无限时域最优值函数的解析表达。

# 3. MPC 收敛于 LQR 的条件

在满足以下条件时，MPC 与 LQR 等价：

  1. 系统为线性且时不变；
  2. 不存在输入与状态约束；
  3. 终端权重取为黎卡提方程的解；
  4. 预测时域趋于无穷；
  5. 权重矩阵保持一致。

此时，MPC 的最优控制律为：

uk=−Kxku_k = -K x_k uk​=−Kxk​

其中，KKK 为 LQR 反馈增益矩阵。因此，LQR = 无约束、无限时域的 MPC。

# 4. 控制策略的本质差异

对比维度 LQR MPC
优化时域 无限时域 有限时域
控制律形式 固定状态反馈 u = -Kx 在线优化得到
增益矩阵 常数 时变
是否滚动优化 否 是
是否显式预测未来 隐式 显式
是否处理约束 否 是
求解方式 离线解析解 在线数值优化
计算复杂度 低 较高
